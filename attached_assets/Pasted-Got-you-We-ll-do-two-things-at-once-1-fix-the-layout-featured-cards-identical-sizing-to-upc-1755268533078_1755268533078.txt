## 1) Scope your demo CSS so it can’t break the app

Your uploaded `index.css` introduces global selectors like `.card`, `.btn`, `:before/:after` that can override the app and produce those big blank bands. Scope them under a class so they only apply on the demo page.

**index.css (add at the top)**

```css
/* ===== DEMO CSS SAFE SCOPE ===== */
.demo-scope .card,
.demo-scope .btn,
.demo-scope .glass,
.demo-scope .badge,
.demo-scope .progress,
.demo-scope .progress::before,
.demo-scope .progress::after {
  all: revert-layer; /* neutralize unexpected cascade */
}

/* Re-apply your demo styles *only* inside .demo-scope */
.demo-scope .buhaus-card { /* <- rename your demo card root to .buhaus-card */
  /* ...move the previous .card rules here... */
}

/* If you prefer not to move declarations right now: */
:root { --et-noop: 0 } /* keeps file valid; real styles moved below */

/* ---- REAL APP UTILITIES (non-invasive) ---- */
.et-grid-eq { display: grid; grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)); gap: 1.25rem; align-items: stretch; }
.et-grid-eq > * { height: 100%; }
.et-card { height: 100%; display: flex; flex-direction: column; }
.et-card__footer { margin-top: auto; }
.et-ellipsis-2 { display:-webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical; overflow: hidden; }
```

> If you used `.card` or `.btn` inside the app, rename those to Tailwind classes or keep them and add `.demo-scope` only around your demo page, e.g. `<div className="demo-scope">…</div>`.

## 2) Make **Featured Elections** use the same card and equal heights

Render featured with the **same ElectionCard** used by “Upcoming”, and force equal heights via the tiny utility classes above.

import { ElectionCard } from "@/components/ElectionCard";

type Props = { items: any[] };

**client/src/components/ElectionCard.tsx (ensure h-full + footer push)**

export function ElectionCard({ election, className = "", variant = "default" }) {
  return (

## 3) Hide the **Data Integrity / Run Auto-fix** widget from normal users

Make it **admin-only**. Easiest: a single env flag and/or role check.

**client/src/lib/useIsAdmin.ts (new)**

```ts
export function useIsAdmin(user?: { role?: string }) {
  // env gate OR role gate. Either will show it.
  const flag = import.meta.env.VITE_ADMIN_FEATURES === "1";
  return flag || (user?.role?.toLowerCase?.() === "admin");
}
```

**client/src/pages/Home.tsx (wrap the widget)**

```tsx
import { useIsAdmin } from "@/lib/useIsAdmin";
// …
const isAdmin = useIsAdmin(currentUser);
// …
{isAdmin && (
  <section className="mt-6">
    <DataIntegrityWidget />  {/* contains Run Auto-fix etc */}
  </section>
)}
```

**Replit env (Secrets):**

```
VITE_ADMIN_FEATURES=0   # public deployment => hidden
```

That completely removes it for the public while keeping it available to staff by setting `VITE_ADMIN_FEATURES=1` (or a real user role when auth lands).

## 4) Consistent sizing with the Virtuoso “Upcoming” grid

Virtuoso already virtualizes the rows; the only trick is to ensure each **item** is `h-full` and the **container** uses equal row sizing. In your Virtuoso `itemContent`, pass `className="h-full"` down to `ElectionCard` (we already support it via `et-card`).

Example (inside your `ElectionGrid`):

```tsx
<VirtuosoGrid
  className="et-grid-eq"
  totalCount={elections.length}
  itemContent={(index) => (
    <ElectionCard election={elections[index]} className="et-card" />
  )}
/>
```
